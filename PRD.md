# 问答社区系统 - 产品需求文档（PRD）

## 系统概述
一个简单的在线问答社区系统，支持用户提问、回答、积分管理等功能。

## 核心功能需求

### 1. 用户系统
- **用户注册**: 邮箱/用户名注册，密码加密存储
- **用户登录**: 支持邮箱或用户名登录，使用 JWT Token
- **个人资料**: 头像、昵称、个人签名、积分等级
- **用户认证**: 基于 Token 的身份验证

### 2. 积分系统
**积分获取规则**:
- 发布问题：-5 积分
- 回答问题：+10 积分
- 回答被采纳：+30 积分
- 问题被点赞：+5 积分（每个赞）
- 回答被点赞：+10 积分（每个赞）

**实现要求**:
- 在相应的操作中自动更新用户积分
- 积分记录可追溯
- 支持积分排行榜

### 3. 点赞功能
**需求**:
- 用户可以点赞问题和回答
- 每个用户对同一个目标只能点赞一次
- 点赞后目标的 votes 数量 +1
- 取消点赞后 votes 数量 -1
- 返回用户是否已点赞的状态
- 点赞操作同步更新积分

### 4. 采纳答案功能
**需求**:
- 只有提问者可以采纳答案
- 每个问题只能采纳一个答案
- 采纳后问题状态变为已解决
- 回答者获得积分奖励（+30 积分）
- 采纳标记在回答卡片中显示

### 5. 问题与回答
**问题发布**:
- 标题、内容（Markdown）、标签
- 支持 Markdown 预览
- 标签多选（最多 5 个）

**问题浏览**:
- 显示问题基本信息、浏览次数、回答数、点赞数
- 支持多种排序方式
- 支持按标签筛选

### 6. 问题列表排序与筛选
**排序方式**:
- **最新**: 按创建时间倒序
- **热门**: 按点赞数倒序
- **未解决**: 筛选 is_solved = false 的问题
- **浏览量**: 按 views 倒序
- **回答数**: 按 answers_count 倒序

**筛选**:
- 按单个或多个标签筛选
- 按问题状态（已解决/未解决）筛选
- 按时间范围筛选（可选）

## 用户界面需求

### 页面结构
1. **首页**: 问题列表、排序、筛选
2. **问题详情页**: 问题信息、回答列表、发布回答表单
3. **发布问题页**: 问题编辑、Markdown 预览、标签选择
4. **用户个人页**: 用户信息、发布的问题、回答、积分等级
5. **排行榜页**: 积分排行、活跃用户
6. **搜索结果页**: 全局搜索功能

## 非功能需求

### 性能要求
- 页面加载时间 < 2s
- API 响应时间 < 500ms
- 支持并发用户数 >= 1000

### 安全性需求
- 密码加密存储（bcrypt）
- SQL 注入防护
- XSS 防护（Markdown 内容过滤）
- CSRF 保护
- API 速率限制

### 可扩展性
- 预留分页逻辑
- 支持缓存机制
- 支持异步操作队列

## 业务流程

### 用户提问流程
1. 用户登录系统
2. 点击发布问题
3. 填写题目、描述、标签
4. 预览内容
5. 提交问题（扣 5 积分）
6. 系统跳转到问题详情页

### 用户回答流程
1. 用户在问题详情页填写回答
2. 预览回答内容
3. 提交回答（获得 +10 积分）
4. 回答显示在列表中

### 采纳答案流程
1. 提问者在问题详情页查看回答
2. 点击「采纳」按钮
3. 系统检查权限（只允许提问者）
4. 标记回答为已采纳
5. 问题状态变为已解决
6. 回答者获得 +30 积分奖励
